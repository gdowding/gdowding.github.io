Setting up for Haskell Development

The recommended way to develop in Haskell is to use the Haskell Tool Stack. Since I'm going functional, I decided to give Nix on OS X a try for installing packages.

Install Nix

The instructions for Getting Nix are pretty easy. I modified slightly to not pipe the output of curl through sh. There is a pretty through article here describing how to install and setup Nix on OS X.

Haskell

Install stack

This will take a while. 

nix-env -i stack

Creating a project

I'm starting a project to manage gitlab merge requests form emacs. I'll be using haskell to interact with gitlab. This is how to start a new project using git and stack. 

cd ~/git
stack new gitlab-mr new-template
cd gitlab-rm
git init
git add .
git commit -m "initial project created using stack"

Building project

The build system needs to be set up by running stack setup. This will download and install the haskell compiler. 

stack setup

Now build will work. 

stack build

Running project

stack exec gitlab-mr-exe
someFunc

Installing Emacs

Now that Haskell is plumbed, I need to set up emacs.

I'll start with haskell-mode, since that seems to be an all-in-one solution.

Since I use Cask, and haskel-mode is a MELPA package. I can update my ~/.emacs.d/Cask file the run cask from that directory. 

;; Haskell
(depends-on "haskell-mode")

cd ~/.emacs.d
cask

Learn Haskell and Implement Project

This will be my first project in haskell. I'm starting with a project that will be nice to have, and that I can work on in my free time. Once it is completed, I can use it to speed up the development time in my other projects. This should give me a chance to evaluate Haskell. If it works out I can use it for more important projects.
